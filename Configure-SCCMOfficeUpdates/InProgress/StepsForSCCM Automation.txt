1. Get Intended version number from user
2. Update "Configuration_UpdateSource.xml" add the "Version" attribute to <Add> element with the value from step 1
2.1 Also update ""Configuration_UpdateTestGroup.xml" with Version and replace any other tokens accordingly.
3. Via PowerShell got to the source share where all the updates will be stored. 
4. Run Setup.exe with /Download against Configuration_UpdateSource.xml, this step will download the bits for required Office Update
5. Create a New Package in SCCM
6. Create a New Program in SCCM under the Package from Step 5
7. Get List of Operating Systems to be supported, and add them to the Program from Step 6, Save the Program
8. Start Content Distribution to the specified Distribution Point or Distribution Point Group and for the specified Collection
9. Start Package Deployment
