Do you want to set up a dynamic solution to deploy Office 365 ProPlus?

A basic deployment can be configured in just a few steps:

1. Generate a dynamic ODT configuration xml file using a computer's existing installation settings using the Generate-ODTConfigurationXML script. This script will detect the languages and Office products that may already be installed.

2. Edit the configuration xml file using the Edit-ODTConfigurationFile script. This function is designed to reduce the effort required to create or makes changes to the configuration file while also reducing the chance of formatting errors in xml.

3. Install Office 365 ProPlus using the Install-OfficeClickToRun script. This PowerShell function will take an existing Office Click-To-Run configuration xml and deploy it using the Office Deployment Tool (ODT).

Do you want to dynamically set a SourcePath by looking at Active Directory for a location?

This Dynamic-UpdateSource function works with the other deployment scripts to provide a way to dynamically set the SourcePath for the Office Click-To-Run configuration xml. The script uses the computers Active Directory site to lookup the SourcePath location from the LookupFilePath. The LookupFilePath is a CSV file that you will need to populate with the Active Directory site and corresponding SourcePath.


Do you want to create a configuration file to deploy specific language packs?

Use the Generate-ODTLanguagePackXML script to create a configuration file that can be used to deploy additional language packs.


Do you want to remove Office Click-To-Run?

This Remove-OfficeClickToRun function will create a configuration xml file and uninstall Office Click-to-Run via the Office Deployment Tool (ODT).


Do you want to remove previous versions of Office?

The Remove-PreviousOfficeInstalls function will remove installations of Office 2013 and older. The Offscrub vbs scripts are used to remove the MSI installations of Office products.
